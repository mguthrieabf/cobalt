Forum moderator


A user with moderator rights for a forum will see red Moderate and Block User buttons against posts and all subsequent comments (except those made by themselves).

Click Moderate. This allows you to edit the post.

Once a post has been moderated the footer of the post will indicate who made the edit, and when.


A user with these rights will see red Moderate and Block User buttons against posts and all subsequent comments (except those made by themselves).

Click Block User. This will prevent that user from posting in this forum (they can still post in other forums and they can still browse posts).


A forum administrator needs to do this.


Forum administrator


Open the Forums main menu item. A user with these rights will see a “Create New Forum” button at the bottom and “Edit” and “Delete” buttons against each forum.

Click Create New Forum.

Complete the details for the new forum – short title, description, type, foreground and background colour.

Click Save to create the new forum and make it available for users.

ABF global payments administrator


Click the Admin main menu item and select ABF payments (found under the Global Payments heading).

To find an individual member statement, click “Lookup Member Statement”, search for the member.

To see a list of all members with non-zero Bridge Credit balances, click “Members with balances”. From this list you can directly open a member statement.


Click the Admin main menu item and select ABF payments (found under the Global Payments heading).

To find an individual organisation statement, click “Lookup Org Statement”, search for the organisation.

To see a list of all organisations with non-zero Bridge Credit balances, click “Orgs with balances”. From this list you can directly open an organisation statement.


Click the Admin main menu item and select ABF payments (found under the Global Payments heading).

Click “Manual adjustment” in the Members or Organisations section of the page depending on where you want to make an adjustment.

Search for the member or organisation.

Input the amount (a positive number increases the account balance) and a description.

The adjustment and description will be included in the relevant statement (your name will appear as the Counterparty).


Click the Admin main menu item and select ABF payments (found under the Global Payments heading).

Click Settlement under the Organisations section.

It is expected that, in almost all cases, the entire balance for an Organisation will be settled. If there are special situations where this is not the case a manual adjustment should be processed

At this stage, the process to actually make the payments through the bank system is manual.

The expected procedure is therefore:

  • View the list of Organisation balances
  • Establish settlements to be made
  • For an account not being settled in full – either manually adjust the balance and include in this listing, or exclude from this listing and process a separate transaction outside the system later
  • Export this listing to csv
  • Use this listing to create a file for processing through the bank
  • Process the actual payments
  • If any of the transactions fail (e.g. the Organisation has provided invalid bank account details) return to this screen and adjust the listing to match what has actually gone through the bank
  • Click Confirm Payment to put the transactions into the Organisation accounts.

    This will provide a confirmation of the transactions generated.

    The transaction can be seen in the relevant Organisation account described as “Settlement from the ABF”

    Click the Admin main menu item and select Stripe Transactions (found under the Global Payments heading).

    This section opens up the Stripe Status report.

    There are three sections that are reported on. The Manual and Auto checkout pending sections highlight items that need investigating. Manual checkout intent is merely a list of places where a user has gone to a payment page but has cancelled before completing the transaction. It might be useful in a scenario where a user believes they have paid for an entry but in fact have not done so.
